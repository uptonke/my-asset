# v10.5.1 Dual Target Blend Delegated Engine

This component generates delegated review drafts from two target-weight sources:

1. Supabase / backup `stock_meta[ticker].target_weight`, displayed in the inventory table as `雲端%`.
2. The v10.5 native target generated by Optimizer Lab from current weights, v5.2 ranking, alpha research, Daily Quant, and Monte Carlo risk context.

The final suggested target weight is a 50/50 blend:

```text
final_suggested_weight_pct = 50% × cloud_target_weight_pct + 50% × v105_native_target_weight_pct
```

The browser-only manual Monte Carlo `%` column is still excluded because it is only created after a manual front-end Monte Carlo run and is not materialized into Supabase or repo JSON for GitHub Actions.

## Target columns

- `current_weight_pct`: current portfolio weight.
- `cloud_target_weight_pct`: Supabase `stock_meta[ticker].target_weight` / `雲端%`.
- `v105_native_target_weight_pct`: Optimizer Lab native target.
- `final_suggested_weight_pct`: blended final suggested target.
- `delta_weight_pct`: final suggested target minus current weight.

## Conflict handling

If the cloud target and v10.5 native target conflict:

- Moderate gap: shrink the drift.
- Major gap: shrink the drift more aggressively.
- Direction conflict: pause the delegated drift for that ticker and show a warning.

This prevents the system from treating a large cloud/native disagreement as a high-confidence trade draft.

## Risk adjustment

Daily Quant and Monte Carlo are used inside the v10.5 native target engine and also produce warnings:

- fragile-node penalties,
- tail-pressure context,
- synthetic-risk availability checks,
- Monte Carlo reference availability checks.

They do not supply the front-end MC% target column in this mode.

## Safety boundary

v10.5.1 remains review-only:

- `trade_signal_enabled: false`
- `execution_permission: false`
- `broker_submission_enabled: false`
- `official_rebalance_enabled: false`
- `auto_trade_enabled: false`
- `not_trade_order: true`
